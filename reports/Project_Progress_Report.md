# 舞蹈姿态评分系统 - 项目进度报告

**日期**: 2026-03-08
**项目名称**: Dance Pose Scoring System

## 1. 项目概述
本项目旨在开发一个智能舞蹈辅助训练系统，通过计算机视觉技术（MediaPipe）实时捕捉用户的舞蹈动作，并与标准参考视频进行对比评分。项目包含 Python 桌面端 (GUI) 和 Web 服务端 (FastAPI)。

---

## 2. 近期重大更新与变动 (Major Updates)

### 2.1 性能与稳定性优化
*   **多线程架构重构 (VideoReader)**
    *   **问题**: 早期版本在主线程读取视频，导致界面卡顿严重，摄像头开启易死机。
    *   **改进**: 引入 `VideoReader` 独立线程类，实现异步视频读取。
    *   **效果**: 彻底解决了 UI 卡顿问题，视频播放流畅，摄像头启动稳定。
*   **视频同步与播放控制**
    *   **问题**: 视频播放速度异常（快放），且用户与参考视频起步不一致。
    *   **改进**: 
        *   在读取线程中加入 FPS 帧率控制，确保按正常速度播放。
        *   实现了 `pause` (暂停) 和 `reset` (重置) 的线程安全逻辑，解决了操作冲突导致的闪退。
        *   引入了 **3秒倒计时** 机制，确保两个视频在 "GO!" 瞬间绝对同步启动。

### 2.2 算法精度提升
*   **评分算法深度优化**
    *   **变动**: 从单纯的“位置距离评分”升级为 **“位置 + 角度”双重评分**。
    *   **细节**: 
        *   引入角度评分权重 (60%)，强制要求关节弯曲度必须准确。
        *   调整权重分配：降低躯干（肩膀/臀部）权重，大幅提升四肢末端（手腕/脚踝）权重。
        *   提高位置匹配的严格度 (`k_decay` 从 2.0 提升至 5.0)。
    *   **效果**: 有效防止了“乱做动作也能拿高分”的情况，评分更具区分度和指导意义。
*   **防抖与状态管理**
    *   **改进**: 增加了评分归零的缓冲时间（30帧），避免因偶尔丢帧导致的评分剧烈闪烁。

### 2.3 用户体验增强
*   **摄像头优化**
    *   **变动**: 默认开启水平镜像翻转 (`cv2.flip`)。
    *   **效果**: 用户看到的画面如同照镜子，动作模仿更符合直觉。
    *   **低延迟模式**: 强制将摄像头分辨率设为 640x480 并禁用缓冲区，显著降低了画面延迟。
*   **实时节奏反馈**
    *   **新增**: 在不影响评分的前提下，增加了时间窗口分析功能。
    *   **效果**: 界面右上角会实时提示 **“快点! (太慢)”** 或 **“慢点! (太快)”**，辅助用户找准节奏。

### 2.4 代码工程化重构
*   **结构调整**
    *   **变动**: 创建 `src` 目录，将单文件代码拆分为 `core` (核心逻辑), `utils` (工具), `ui` (界面) 模块。
    *   **效果**: 消除了 `gui_app.py` 和 `backend_app.py` 之间的代码重复，提升了项目的可维护性和扩展性。
